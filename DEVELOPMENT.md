# Dotfiles — Personal Configuration Management

> All personal configs in one place. Bootstrap any new machine from Valhalla.

## Overview

Centralized dotfiles management inside the Valhalla monorepo. Shell configs, editor settings, git config, SSH config, tool configs, and a bootstrap script to set up any new machine (Linux, macOS, or code-server) with a single command.

Previously lived in a separate GitHub repo (meisterveda/dotfiles) which was mostly a Docusaurus scaffold. This replaces it with actual dotfile management using GNU Stow.

**Status:** Future project (scaffolding now)

## Strategy

### GNU Stow

Each config category is a **stow package** — a directory whose contents mirror the target home directory structure. `stow` creates symlinks from `~/` to the files in this repo.

```bash
# Example: stow the zsh package
cd apps/dotfiles
stow -t ~ zsh
# Creates symlink: ~/.zshrc -> /path/to/valhalla/apps/dotfiles/zsh/.zshrc
```

This means:
- Edit files in the repo, changes are live immediately (symlinks)
- `git diff` shows config changes
- New machine setup is just `git clone` + `stow`

## Bootstrap Script

`bootstrap.sh` sets up a fresh machine end-to-end:

```bash
curl -fsSL https://raw.githubusercontent.com/meisterveda/valhalla/main/apps/dotfiles/bootstrap.sh | bash
# Or from local clone:
./apps/dotfiles/bootstrap.sh
```

### What It Does

1. **Detect OS** (Ubuntu/Debian, macOS, Arch, code-server)
2. **Install system packages:**
   - zsh, git, curl, wget, jq, htop, tree, ripgrep, fd-find, bat, eza
   - stow, tmux, neovim
3. **Install dev tools:**
   - nvm + Node.js LTS
   - Rust (rustup)
   - Go
   - Python + pip
4. **Install CLI tools:**
   - kubectl, helm, k9s
   - terraform
   - gh (GitHub CLI), glab (GitLab CLI)
   - mc (MinIO client)
   - claude (Claude Code CLI)
   - docker CLI
5. **Clone Valhalla** (if not already cloned)
6. **Stow all packages:**
   ```bash
   cd apps/dotfiles
   stow -t ~ zsh bash git ssh nvim vscode tmux starship scripts claude
   ```
7. **Set zsh as default shell**
8. **Install zsh plugins** (zsh-autosuggestions, zsh-syntax-highlighting, zsh-completions)
9. **Configure git** (prompt for name/email if not set)
10. **Set up SSH keys** (generate if missing, prompt to add to GitLab/GitHub)

### Machine Profiles

The bootstrap script detects the machine type and adjusts:

| Profile | Extra Tools | Notes |
|---------|-------------|-------|
| `workstation` | Full dev tools, Docker, K8s tools | Primary dev machine |
| `server` | Minimal — just shell + git + k8s tools | K8s nodes, VMs |
| `code-server` | Dev tools, no Docker | code.home.meisterveda.com |
| `macos` | Homebrew-based installs | MacBook |

## Makefile

```makefile
STOW_PACKAGES = zsh bash git ssh nvim vscode tmux starship scripts claude

install:    ## Full bootstrap
	./bootstrap.sh

stow:       ## Symlink all packages
	stow -t ~ $(STOW_PACKAGES)

unstow:     ## Remove all symlinks
	stow -t ~ -D $(STOW_PACKAGES)

restow:     ## Re-symlink (fix conflicts)
	stow -t ~ -R $(STOW_PACKAGES)
```

## Notes

- **No secrets in dotfiles.** SSH keys are generated by bootstrap, not stored. Vault tokens are fetched at runtime.
- **Stow conflicts:** If a target file already exists and isn't a symlink, stow will refuse. Bootstrap handles this by backing up existing files first.
- **Testing:** Bootstrap script should be idempotent — safe to run multiple times.
- **No Nx integration needed.** Dotfiles are shell-managed, not built/compiled. No project.json.
